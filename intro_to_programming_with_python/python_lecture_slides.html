üßÆ The Order Matters - Operations and Precedence

When calculating team sizes, order of operations matters:

# Calculating teams with different approaches
total_students = 127
base_team_size = 4
extra_members = 3

# Wrong way (without parentheses)
result = total_students / base_team_size + extra_members
print(result)  # 34.75 (not what we want!)

# Right way (with parentheses)
result = total_students / (base_team_size + extra_members)
print(result)  # 18.14 (makes more sense!)

üõ†Ô∏è Building Our Own Team Generator

Let's combine everything we've learned to create a simplified version of the team generator tool we used earlier!

import random ## Part 1

def create_teams(student_names, team_size=4):
    """
    Create random teams from a list of student names.
    This is our algorithm design in action!
    """
    # Step 1: Make a copy and shuffle (randomization)
    students = student_names.copy()
    random.shuffle(students)
    
    # Step 2: Create teams (decomposition)
    teams = []
    team_number = 1
    
    # Step 3: Group students (pattern recognition)
    for i in range(0, len(students), team_size):
        team = students[i:i + team_size]
        teams.append({
            'number': team_number,
            'members': team
        })
        team_number += 1
    
    return teams

üéØ Team Display Function

def display_teams(teams): ## Part 2 (paste below Part 1)
    """
    Display teams in a nice format (abstraction - hiding complexity)
    """
    print("\n" + "="*50)
    print("üéØ TEAM ASSIGNMENTS")
    print("="*50)
    
    for team in teams:
        print(f"\nüìã Team {team['number']}:")
        for i, member in enumerate(team['members'], 1):
            print(f"   {i}. {member}")
        print(f"   Total: {len(team['members'])} members")
    
    print(f"\nüìä Summary: {len(teams)} teams created!")

üß† Computational Thinking in Action:

‚úÖ Decomposition: Broke problem into shuffle‚Üígroup‚Üídisplay
‚úÖ Pattern Recognition: Same grouping process for each team
‚úÖ Abstraction: Functions hide complex details
‚úÖ Algorithm Design: Repeatable process that works for any names/size
